# Idea Crucible

A personal anti procrastination app for chronic brainstormers. Jot raw ideas into a sandbox, promote at most two into the Arena, break each into 15 minute micro tasks, and let the app nag you. Any active project with zero progress for 14 days hits a decision portal: bury it, revive it by doing a task right now, or send it on a one time sabbatical. There is also an Android home screen widget showing the project closest to the cemetery.

Built with Expo (React Native), local notifications, on device storage, and a native Android widget. No server, no account, no cost beyond your phone.

## Run it (Windows, Android phone)

You have two ways in. The first is the most reliable because Expo picks dependency versions that match the current SDK.

### Option A, recommended: fresh project, copy the source in

1. Install Node LTS and Android Studio on your PC.
2. Create a blank project and move into it:
   ```
   npx create-expo-app@latest idea-crucible --template blank-typescript
   cd idea-crucible
   ```
3. Add the libraries (this pins versions correctly for your SDK):
   ```
   npx expo install expo-font expo-notifications expo-splash-screen expo-status-bar @react-native-async-storage/async-storage @expo/vector-icons @expo-google-fonts/archivo-black @expo-google-fonts/space-grotesk @expo-google-fonts/space-mono react-native-android-widget
   ```
4. Copy these from this download into the new project, replacing what is there: the entire `src` folder, `App.tsx`, `index.js`, `app.json`, and the `assets` folder. In `app.json`, keep the `plugins` block and set `android.package` to whatever you like.
5. Continue to "Get it on your phone" below.

### Option B: use these files directly

1. Unzip, then inside the folder run:
   ```
   npm install
   npx expo install --fix
   ```
   The `--fix` step realigns native dependency versions to your Expo SDK. Do not skip it. The versions in `package.json` are a starting point, not gospel.
2. Continue below.

### Get it on your phone

This app uses notifications and a widget, both of which are native, so it runs in a development build rather than Expo Go.

1. Enable Developer options and USB debugging on your Android phone, then plug it into the PC.
2. From the project folder:
   ```
   npx expo run:android
   ```
   This compiles a development build and installs it on the connected phone. The first build takes a while.
3. After it is installed, start the dev server with `npm start`. From then on your code changes reload instantly, and you only rebuild when you change native config (like the widget settings in `app.json`).

No paid account is needed for Android. If you ever prefer a cloud build instead of plugging in over USB, run `npx eas build --profile development --platform android` and install the APK it produces.

### Add the widget

Long press the home screen, choose Widgets, find Idea Crucible, and drop it on. It refreshes hourly on its own and immediately whenever you complete a task while the app is open.

## Optional: real AI micro tasks

Out of the box, promoting an idea generates sensible generic starter tasks with no setup. If you want tasks generated by a model from the idea itself, paste an Anthropic API key into `src/config.ts`. That key ships inside your personal build, which is fine for an app only you install. Never do that for an app you hand to other people.

## How it works

The core rule lives in the engine: completing a micro task is the only thing that resets the 14 day clock. Opening the app does not count. When you complete a task, the app cancels the old day 10, 13, and 14 reminders and schedules fresh ones from that moment.

- Day 10: a nudge.
- Day 13: a 24 hour warning.
- Day 14: the project locks and the decision portal opens with three choices.

## File map

Logic (UI free, reusable):
- `src/types.ts` data model
- `src/constants.ts` the slot cap and the 10 / 13 / 14 timeline
- `src/store.ts` storage over AsyncStorage
- `src/arena.ts` pure rules: slots, days since execution, decision point
- `src/notifications.ts` local notification scheduling
- `src/engine.ts` orchestration: promote, complete, bury, revive, sabbatical
- `src/microsteps.ts` AI generation plus the offline fallback
- `src/widgetView.ts` what the widget shows
- `src/widget.tsx` push an instant widget refresh
- `src/widgetTaskHandler.tsx` background widget refresh
- `src/index.ts` public exports, initEngine, nag window helpers
- `src/config.ts` optional API key

UI (the look and screens):
- `src/ui/theme.ts` colors, fonts, the offset shadow. Change the whole look here.
- `src/ui/ChunkyCard.tsx`, `ChunkyButton.tsx`, `ModalFrame.tsx`, `ScreenTitle.tsx` building blocks
- `src/ui/nav.tsx` tiny navigation context
- `src/ui/AppShell.tsx` tabs, bottom nav, modal host
- `src/ui/HomeScreen.tsx` the four zone dashboard
- `src/ui/PromoteModal.tsx`, `ProjectModal.tsx`, `DecisionModal.tsx` the flows
- `src/ui/CrucibleScreen.tsx`, `HistoryScreen.tsx`, `ProfileScreen.tsx` the other tabs

Entry and config: `App.tsx`, `index.js`, `app.json`, `package.json`, `tsconfig.json`, `babel.config.js`, `assets/`.

## Known edges

- The weekly bars and stats are real, computed from task completion times.
- History is read only for now. Buried and sabbatical ideas are listed but not yet revivable from that screen.
- Some Android phones delay background notifications to save battery. Exempt the app from battery optimization if your nags arrive late.
